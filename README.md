# UK Telecoms Churn Prediction

End-to-end machine learning project for proactive broadband churn management.

The repository includes:
- data ingestion and validation
- feature engineering
- LightGBM training + calibration
- survival analysis and uplift experimentation
- explainability, drift monitoring, and batch scoring
- FastAPI serving and Airflow-ready orchestration DAGs

## 1) Repository layout

```text
conf/            # Configs (paths, model params, monitoring thresholds)
data/            # raw, processed, and feature data locations
docker/          # Dockerfiles + docker-compose stack
notebooks/       # 01–08 analysis and modelling narrative
outputs/         # model artefacts, scored outputs, figures, reports
pipelines/       # Airflow DAGs (daily scoring, drift monitor, weekly retrain)
src/churn/       # package code (data, features, models, api, monitoring, etc.)
tests/           # unit + data-quality + integration test scaffolding
```

## 2) Tech stack

- Python 3.10+
- Dependency/runtime management: `uv`
- Data: `pandas`, `duckdb`, `pyarrow`
- Modelling: `lightgbm`, `scikit-learn`, `lifelines`, `optuna`
- Tracking/monitoring: `mlflow`, `evidently`
- API: `fastapi`, `uvicorn`
- Quality: `pytest`, `ruff`, `mypy`

## 3) Quickstart

### Prerequisites

- Python 3.10+
- `uv` installed

### Install

```bash
make install-dev
```

Optional extras:

```bash
make install-api
make install-browsers
```

## 4) Data and configuration

Main paths are configured in `conf/config.yaml`:
- raw data: `data/raw`
- processed DuckDB: `data/processed/churn.db`
- feature snapshots: `data/features`
- outputs: `outputs/`

Expected raw inputs:
- `data/raw/calls.csv`
- `data/raw/cease.csv`
- `data/raw/customer_info.parquet`
- `data/raw/usage.parquet`

Temporal splits are also controlled in `conf/config.yaml` (`train_end_date`, `val_end_date`, `test_end_date`).

## 5) Core workflow

Run the project pipeline with Make targets:

```bash
make setup-db      # ingest + schema checks
make features      # build feature store snapshots
make train         # train/tune model and log artefacts
make evaluate      # evaluation plots/metrics
make score         # batch scoring to outputs/scored_customers.csv
make explain       # SHAP explainability outputs
make monitor       # drift report
```

## 6) Notebooks

Notebook storyline in `notebooks/`:
- `01_eda.ipynb` — exploratory analysis
- `02_data_quality.ipynb` — leakage/data checks
- `03_feature_engineering.ipynb` — feature construction
- `04_modelling.ipynb` — baseline/champion modelling
- `05_evaluation.ipynb` — performance and business metrics
- `06_explainability.ipynb` — global/local explanations
- `07_survival_uplift.ipynb` — survival + uplift-oriented analysis
- `08_mlops_demo.ipynb` — operational demo

Export notebook reports:

```bash
make reports
make reports-clean
```

## 7) API serving

Start API:

```bash
make serve
```

Key endpoints:
- `GET /health`
- `POST /predict`
- `POST /batch-score`

Production-style run:

```bash
make serve-prod
```
